# Issue #12：Unsupported parameter: 'max_tokens' is not supported with this model

**来源**：https://github.com/cprobe/catpaw/issues/12  
**提交者**：@qizai123  
**日期**：2026-03-10  
**状态**：已修复

---

## 问题描述

调用公司内部 GPT-5.2 代理时，请求被拒绝：

```text
Unsupported parameter: 'max_tokens' is not supported with this model. Use 'max_completion_tokens' instead
```

用户已经在配置里改成 `max_completion_tokens = 4000`，但 agent 仍然报同样错误。

## 环境信息

- 版本：catpaw-v0.13.0-linux-amd64
- 模型：gpt-5.2
- 部署：公司内部 GPT 代理（OpenAI-compatible `/v1/chat/completions`）

---

## 分析结论

这是 **agent 侧的兼容性 bug**，不是单纯的配置问题。

### 根因

1. `ModelConfig` 只定义了 `max_tokens`，不识别 `max_completion_tokens`
2. `AIConfig.applyDefaults()` 会在 `max_tokens` 未设置时自动补默认值 `4000`
3. OpenAI-compatible client 构造请求体时写死发送 `max_tokens`
4. 即使用户在配置里写了 `max_completion_tokens`，配置层也不会接住，最终仍会发出默认的 `max_tokens=4000`

### 代码定位

- `config/config.go`
  - `ModelConfig` 仅包含 `MaxTokens`
  - `applyDefaults()` 在未配置时回填 `MaxTokens = 4000`
- `diagnose/aiclient/failover.go`
  - `newChatClient()` 仅向 client 传递 `MaxTokens`
- `diagnose/aiclient/client.go`
  - `buildRequestPayload()` 固定写入 `body["max_tokens"]`

### 为什么用户改了配置仍然无效

`max_completion_tokens` 原先不在配置结构体中，因此 TOML 加载后不会进入 `ModelConfig`。随后默认值逻辑又把 `max_tokens` 补成了 `4000`，请求体继续发送旧字段，所以服务端仍然报错。

---

## 修复内容

### 1. 配置层新增 `max_completion_tokens`

- `ModelConfig` 新增 `MaxCompletionTokens`
- 兼容两种配置：
  - 旧模型继续使用 `max_tokens`
  - GPT-5 风格模型可使用 `max_completion_tokens`

### 2. 默认值逻辑改为避免误补旧字段

- 只有在 `max_tokens` 和 `max_completion_tokens` 都未配置时，才回填默认 `max_tokens = 4000`
- 若用户显式配置了 `max_completion_tokens`，不再补 `max_tokens`

### 3. 请求体只发送一个 token 限制字段

- 若配置了 `max_completion_tokens`，请求体只发该字段
- 否则回退发送 `max_tokens`
- 同时清理 `extra_body` 中同名冲突字段，避免双字段同时出现

### 4. 补充测试

- 校验 TOML 能正确加载 `max_completion_tokens`
- 校验默认值逻辑不会覆盖 `max_completion_tokens`
- 校验请求体优先发送 `max_completion_tokens`
- 校验不会同时发送 `max_tokens` 和 `max_completion_tokens`

---

## 兼容性说明

- 旧配置 `max_tokens` 继续可用
- 新配置 `max_completion_tokens` 对 GPT-5 风格 OpenAI-compatible 代理生效
- 两者同时配置时，client 优先发送 `max_completion_tokens`

---

## 建议配置

对于 GPT-5 风格模型，建议这样配置：

```toml
[ai.models.gpt5]
base_url = "http://llmgw.xxx.com/v1"
api_key = "sk-xxxx"
model = "gpt-5.2"
max_completion_tokens = 4000
```

旧模型保持原有 `max_tokens` 配置即可。
